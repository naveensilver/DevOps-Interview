# 🔥 Prometheus Interview Prep (Structured Answers)

<details>
<summary><b>Q1: What is Prometheus and how is it different from traditional monitoring tools?</b></summary>

<br>

Prometheus is an open-source monitoring and alerting tool that collects time-series metrics using a pull-based model and stores them in its own TSDB.

**Why it is used**

Traditional monitoring tools mostly depend on push-based systems and are not well-suited for dynamic environments like Kubernetes.

Prometheus solves this by:

* Automatically discovering services
* Collecting real-time metrics
* Allowing flexible querying using labels

**How it works in production (Flow)**

```text
Application / Exporter → /metrics endpoint → Prometheus (scrapes) → TSDB → Alertmanager → Grafana
```

* Applications expose metrics via `/metrics`
* Prometheus scrapes metrics at defined intervals
* Stores data in time-series format
* Alerts are triggered using rules
* Grafana is used for visualization

**Real-time Example (Production Style Answer)**

“In our Kubernetes environment, we used Prometheus to monitor pod-level and node-level metrics. We deployed Node Exporter and kube-state-metrics, and Prometheus used Kubernetes service discovery to automatically scrape all the targets.

We created dashboards in Grafana to monitor CPU, memory, and API latency, and set up alerts for high error rates and pod restarts using Alertmanager.”

**Strong Closing Line**

“So overall, Prometheus is highly suitable for cloud-native and microservices environments because of its pull-based model, dynamic service discovery, and powerful querying capabilities.”
